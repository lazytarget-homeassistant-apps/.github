# Contribution Guidelines

## How to contribute

1. Fork this repository and make your changes on your fork an a branch you like (I recommend to create a branch per feature).
2. Create a draft PR with your changes
3. Await for the automatic checks to run and publish the PR after a successful result.
4. After your changes have been reviewed and approved your changes can be merged into the `main` branch and possibly a release can be created.

## Requirements to get Pull Requests accepted

* Passes the build (Including CI/CD, tests, shellcheck, markdownlint and others...)
* Review from a maintainer
